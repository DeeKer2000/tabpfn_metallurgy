---

## 1. 项目概述

本项目开发了一套铜熔炼智能配矿优化系统，技术路线为：

```
矿料数据 → 合成炉料采样(3000组) → FactSage 8.3热力学批量计算
→ TabPFN代理模型训练(29个目标) → NSGA-II单目标优化 → 最优配矿方案
```

核心创新：用 FactSage 作为"数据生成器"解决冶金领域小样本问题，用 TabPFN（Nature 2025 预训练表格基础模型）作为代理模型实现毫秒级预测，将代理模型嵌入NSGA-II优化器实现智能配矿。

## 3. 程序运行方法

### 环境要求

- Conda环境：`tabpfn`，Python 3.12 + CUDA 11.8
- 关键依赖：tabpfn 8.0.8、tabpfn-extensions 0.4.2、pymoo
- **所有命令从项目根目录运行**

### 运行配矿优化

```bash
conda run -n tabpfn --env PYTHONIOENCODING=utf-8 python scripts/optimization/nsga2_optimization.py
```

**预计耗时**：10~20分钟（取决于GPU）

**优化设置**：
- 目标：最小化渣含Cu（wt%）
- 约束：Fe/SiO₂ ∈ [1.15, 1.30]，冰铜品位 ∈ [58, 60] wt%
- 算法：NSGA-II（种群100，迭代200代，早停止patience=30）

**输出文件**（位于 `experiments/3000_fixed_temp/optimization/`）：
- `optimization_pareto_all.csv` — 所有可行解
- `optimization_results.csv` — 代表性方案
- `figures/fig_pareto_front.png/pdf` — 优化过程图
- `figures/fig_optimization_comparison.png/pdf` — 对比柱状图

### 运行图表生成（从CSV重新生成图表）

```bash
conda run -n tabpfn python scripts/optimization/plot_optimization.py
```

### 其他常用命令

```bash
# 生成合成炉料样本
conda run -n tabpfn python scripts/data_processing/generate_samples.py

# FactSage结果转CSV
conda run -n tabpfn python scripts/data_processing/convert_factsage_to_csv.py

# 训练TabPFN模型
conda run -n tabpfn python scripts/training/train_tabpfn.py

# SHAP分析
conda run -n tabpfn python scripts/analysis/shap_analysis.py

# Baseline对比（RF/XGBoost/GB/MLP）
conda run -n tabpfn python scripts/analysis/compare_baselines.py
```

## 4. 有了人工配矿数据后的对比方法

### 4.1 准备人工方案数据

你需要准备人工配矿方案的 **11种矿料配比** 和 **SiO₂熔剂比例**。格式示例：

```python
# 人工方案的11种矿料配比（需归一化，总和=1）
# 矿料顺序：炎鑫, 汇祥永金, 嘉能可ET, 金源矿冶, 阿舍勒, 路源, 外矿BOZ, 外矿AKT, 和鑫, 渣精矿, 白银
manual_ratios = [0.05, 0.12, 0.20, 0.15, 0.03, 0.08, 0.10, 0.08, 0.05, 0.02, 0.12]

# SiO₂熔剂添加比例（0~0.15）
manual_sio2_flux = 0.02
```

如果有多个历史人工方案，可以准备多组。

### 4.2 用TabPFN预测人工方案的冶金指标

在Python中执行：

```python
import numpy as np
from scripts.optimization.nsga2_optimization import ratios_to_features, load_models, ORE_COMPOSITION

# 加载模型（只需 slag_Cu_pct 和 matte_Cu_pct）
models = load_models()

# 转换为16维输入
features = ratios_to_features(np.array(manual_ratios), manual_sio2_flux)

# 预测
slag_cu = models['slag_Cu_pct'].predict(features.reshape(1, -1))[0]
matte_cu = models['matte_Cu_pct'].predict(features.reshape(1, -1))[0]
fesio2 = models['slag_FeO_SiO2_ratio'].predict(features.reshape(1, -1))[0]

print(f"渣含Cu: {slag_cu:.4f} wt%")
print(f"冰铜品位: {matte_cu:.2f} wt%")
print(f"Fe/SiO₂: {fesio2:.3f}")
```

### 4.3 修改对比图脚本

编辑 `scripts/optimization/plot_optimization.py` 中的 `plot_comparison_bar()` 函数，将 `manual` 字典替换为真实数据：

```python
manual = {
    'slag_Cu_pct': slag_cu,      # 替换为预测值
    'matte_Cu_pct': matte_cu,     # 替换为预测值
    'slag_FeO_SiO2_ratio': fesio2, # 替换为预测值
}
```

然后重新运行：

```bash
conda run -n tabpfn python scripts/optimization/plot_optimization.py
```

### 4.4 对比逻辑说明

| 对比项 | 人工方案 | 优化方案 | 期望结果 |
|--------|---------|---------|---------|
| 渣含Cu | TabPFN预测值 | 优化器最小化目标 | 优化 < 人工 |
| 冰铜品位 | TabPFN预测值 | 约束在58~60% | 两者接近 |
| Fe/SiO₂ | TabPFN预测值 | 约束在1.15~1.30 | 两者在范围内 |

由于人工方案和优化方案都通过同一个TabPFN模型预测，对比是公平的（同一模型尺度）。

## 6. 已知问题与注意事项

### 编码问题

- 输入数据文件是 **GBK编码**，CSV输出是 **utf-8-sig**
- `conda run` 在Windows上会用GBK重编码头输出，含`₂`等Unicode字符会报错。解决：设置 `PYTHONIOENCODING=utf-8`
- matplotlib中文标签会导致进程挂起（CJK字体缺失），图表脚本改用英文标签

### TabPFN模型加载

- 保存的模型中 `init_params.json` 的 `model_path` 是相对路径
- `nsga2_optimization.py` 中的自定义加载函数已解决：手动解压归档 → 替换 `model_path` 为绝对路径 → 重建 `TabPFNRegressor`

### pymoo约束处理

- pymoo的罚函数法允许部分不满足约束的解进入结果集
- `select_representative_solutions()` 会在后处理中按约束过滤

## 7. 如何添加更多优化目标

当前程序是1个目标（min 渣含Cu）+ 4个约束。如果需要同时优化多个指标，按以下步骤修改 `scripts/optimization/nsga2_optimization.py`。

### 7.1 确定目标和约束的划分

| 指标 | 建议作为 | 原因 |
|------|---------|------|
| 渣含Cu | 目标（已有） | 越小越好，无固定范围 |
| 冰铜品位 | 约束（已有） | 58~60%，有明确范围 |
| Fe/SiO₂ | 约束（已有） | 1.15~1.30，有明确范围 |
| 配矿铜品位（混合Cu%） | 可改为目标 | 如果希望越高越好 |
| 有害元素（As、Sb等） | 约束 | 有环保上限，不适合做目标 |
| 渣中Cu损失率 | 可加为目标 | 越小越好 |
| 成本 | 可加为目标 | 越小越好（需要成本模型） |

**原则**：有明确范围的指标做约束，"越大越好"或"越小越好"的做目标。

### 7.2 修改步骤（以加"配矿铜品位最大化"为例）

**Step 1：修改 Problem 类的 `__init__`**

```python
# 旧：1个目标，4个约束
n_obj = 1
n_ieq_constr = 4

# 新：2个目标，4个约束
n_obj = 2
n_ieq_constr = 4
```

**Step 2：修改 `_evaluate` 中的 F 矩阵**

```python
# 旧：F shape (n, 1)
F = np.full((n, 1), 1e6)

# 新：F shape (n, 2)
F = np.full((n, 2), 1e6)
```

**Step 3：在 `_evaluate` 的批量预测部分，计算新目标**

```python
# 假设你要最大化混合炉料铜品位（blend_Cu）
# 在批量预测之后添加：
for idx, i in enumerate(valid_indices):
    # 原有目标：min 渣含Cu
    F[i, 0] = float(pred_slag_cu[idx])
    # 新目标：max 配矿铜品位（取负号变为最小化）
    F[i, 1] = -float(blend_Cu[idx])  # blend_Cu 需要从配比计算
```

**Step 4：修改 `extract_results` 中的结果提取**

```python
# F_opt 现在是 (n, 2)
slag_cu = float(F_opt[i, 0])
blend_cu = -float(F_opt[i, 1])  # 转回正值
```

**Step 5：如果新目标替换了某个约束，删除对应约束**

```python
# 例如把冰铜品位从约束改为目标：
n_ieq_constr = 2  # 只剩 Fe/SiO₂ 的 2 个约束
# 同时删除 G[i, 2] 和 G[i, 3]（matte_Cu 约束）
# 在 F 中加入 matte_Cu 作为第 2 个目标
```

### 7.3 添加有害元素约束示例

有害元素（As、Sb等）适合做约束（有上限），不需要改目标数量：

```python
# 在 check_metallurgical_constraints 中添加：
if not (as_content <= 0.1):   # 砷含量上限 0.1%
    return False
if not (sb_content <= 0.05):  # 锑含量上限 0.05%
    return False

# 混合炉料中有害元素含量按配比加权计算：
as_content = sum(r_i * As_i for r_i, As_i in zip(ratios, As_list))
sb_content = sum(r_i * Sb_i for r_i, Sb_i in zip(ratios, Sb_list))
```

### 7.4 目标数与算法选择

| 目标数 | 算法 | 结果形式 |
|--------|------|---------|
| 1个 | NSGA-II（当前） | 单一最优解 |
| 2~3个 | NSGA-II | Pareto前沿（一组权衡方案） |
| 4个以上 | NSGA-III、MOEA/D | 高维Pareto前沿 |

切换算法只需改 `run_optimization()` 中的导入：
```python
# 多目标 → NSGA-II（已默认使用）
from pymoo.algorithms.moo.nsga2 import NSGA2
algorithm = NSGA2(...)
```
