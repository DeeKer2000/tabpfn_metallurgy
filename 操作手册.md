## 2. 数据生成（合成炉料样本）

**脚本**：`scripts/data_processing/generate_samples.py`

**作用**：基于 11 种矿料的化学组成，随机生成 N 组配料方案，供 FactSage 热力学计算使用。

### 2.1 运行命令

```bash
conda run -n tabpfn python scripts/data_processing/generate_samples.py
```

### 2.2 输入文件

| 文件 | 说明 |
|------|------|
| `data/矿源.xlsx` | 11 种矿料的化学组成（干基，wt%），含 Cu、S、Fe、SiO₂、Pb、Zn、MgO、Al₂O₃、CaO、Ni、Sb、Bi、As 共 13 个组分 |

### 2.3 可调参数

编辑脚本顶部的参数配置区：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `N_SAMPLES` | 30000 | 生成样本数量 |
| `RANDOM_SEED` | 42 | 随机种子（设为 None 则每次不同） |
| `N_ORE_MIN` | 2 | 每组选用的最少矿种数 |
| `N_ORE_MAX` | 6 | 每组选用的最多矿种数 |
| `SIO2_ADD_MIN` | 0.0 | SiO₂ 熔剂添加量下限（占矿石质量%） |
| `SIO2_ADD_MAX` | 15.0 | SiO₂ 熔剂添加量上限 |
| `TEMP_FIXED` | None | 固定温度（如 1300.0），None 则随机 |
| `TEMP_MIN` | 1190.0 | 温度下限（℃），TEMP_FIXED=None 时生效 |
| `TEMP_MAX` | 1350.0 | 温度上限（℃），TEMP_FIXED=None 时生效 |
| `PRESSURE` | 1.0 | 压强（atm），铜熔炼通常为 1 atm |

### 2.4 冶金成分约束

生成的样本会自动过滤，确保混合炉料成分满足以下约束：

| 成分 | 下限 | 上限 |
|------|------|------|
| Cu | 14.0 wt% | 26.0 wt% |
| S | 8.0 wt% | 38.0 wt% |
| Fe | 8.0 wt% | 36.0 wt% |
| SiO₂ | 3.0 wt% | 25.0 wt% |
| Fe/SiO₂ 比 | 0.8 | 4.0 |

### 2.5 输出文件

| 文件 | 说明 |
|------|------|
| `data/合成炉料样本.xlsx` | Excel 格式，含所有样本的完整信息 |
| `data/合成炉料样本.txt` | Tab 分隔文本（UTF-8 BOM），**FactSage 批量计算的输入文件** |

**输出列**：序号、温度(℃)、压强(atm)、11 种矿料配比(wt%)、SiO₂ 原矿含量、SiO₂ 添加量、最终 13 种成分含量(wt%)。

### 2.6 注意事项

- 样本数量大时，约束过滤会导致多次重试，脚本会报告实际尝试次数
- 矿料配比归一化后总和为 100%，SiO₂ 熔剂作为额外质量添加
- `.txt` 文件是 FactSage 批量计算的输入格式

## 3. FactSage 批量计算（手动）

**此步骤在 FactSage 8.3 软件中手动完成，非脚本操作。**

### 3.1 操作流程

1. 打开 FactSage 8.3 → Equilib 模块
2. 加载 `data/合成炉料样本.txt` 作为批量计算输入
3. 选择数据库：FToxid（渣）+ FSstel（冰铜/金属）
4. 设置计算条件：温度 1190~1350℃，压强 1 atm
5. 选择输出相：Matte（冰铜）、Slag-liq#1（渣液相）、gas_real（气相）
6. 运行批量计算
7. 将结果逐个保存为 `result_XXXXX.txt` 格式

### 3.2 结果存放

将 FactSage 计算结果放入：

```
data/3000_fixed_temp/factsage_results/result_00000.txt
data/3000_fixed_temp/factsage_results/result_00001.txt
...
data/3000_fixed_temp/factsage_results/result_XXXXX.txt
```

### 3.3 结果文件格式

每个 `result_XXXXX.txt` 包含热力学平衡计算结果，主要包含三个相的信息：

- **Matte**（冰铜）：质量(g) + 各组分 wt%（Cu、Fe、S、Ni、Zn、Pb、As）
- **Slag-liq#1**（渣液相）：质量(g) + 各组分 wt%（Cu₂O、FeO、SiO₂、Al₂O₃、CaO、MgO、ZnO）
- **gas_real**（气相）：质量(g) + 各组分 mol 分数（SO₂、S₂、SO、SSO、SO₃、Zn、PbS、CuS、Pb、Sb、As）

## 5. TabPFN 模型训练

**脚本**：`scripts/training/train_tabpfn.py`

**作用**：对 29 个输出目标分别训练 TabPFN 回归模型。TabPFN 使用 in-context learning（不进行梯度下降），训练极快（秒级到分钟级）。

### 5.1 运行命令

```bash
conda run -n tabpfn python scripts/training/train_tabpfn.py
```

### 5.2 输入文件

| 文件 | 说明 |
|------|------|
| `data/3000_fixed_temp/ml_dataset.csv` | ML 训练数据集 |
| `TabPFN-main/models/tabpfn-v3-regressor-v3_20260417_mediumdata.ckpt` | TabPFN v3 预训练权重 |

### 5.3 可调参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `DATA_PATH` | `data\3000_fixed_temp\ml_dataset.csv` | 数据文件路径 |
| `MODEL_PATH` | `TabPFN-main\models\...ckpt` | 预训练权重路径 |
| `RESULTS_DIR` | `experiments\3000_fixed_temp\tabpfn` | 结果输出目录 |
| `DEVICE` | `cuda` | 计算设备：`cuda` 或 `cpu` |
| `TEST_SIZE` | 0.2 | 测试集比例 |
| `RANDOM_STATE` | 42 | 随机种子（保证可复现） |
| `BATCH_SIZE` | 1000 | 预测分批大小（防 OOM） |

### 5.4 训练流程

1. 加载 `ml_dataset.csv`
2. 按 80/20 划分训练集/测试集
3. 对 29 个目标逐一训练：
   - 创建 `TabPFNRegressor(ignore_pretraining_limits=True)`
   - 调用 `.fit(X_train, y_train)` — 将训练数据编码进 Transformer 上下文
   - 分批预测测试集
   - 计算 R²、MAE、RMSE
   - 保存模型为 `.tabpfn_fit` 文件
4. 生成评估结果 CSV 和散点图

### 5.5 输出目录

```
experiments/3000_fixed_temp/tabpfn/{YYYYMMDD_HHMM}/
├── evaluation_results.csv          # 29个目标的评估指标
├── saved_models/                   # 29个模型文件
│   ├── matte_Cu_pct.tabpfn_fit
│   ├── matte_Fe_pct.tabpfn_fit
│   ├── ...
│   └── gas_mass_g.tabpfn_fit
└── scatter_true_vs_pred.png        # 真实值 vs 预测值散点图
```

**evaluation_results.csv 列说明**：

| 列名 | 含义 |
|------|------|
| `target` | 目标变量名 |
| `R²` | 决定系数（越接近 1 越好） |
| `MAE` | 平均绝对误差 |
| `RMSE` | 均方根误差 |
| `mean_val` | 目标变量均值 |
| `MAE_pct` | MAE 占均值的百分比 |

### 5.6 注意事项

- TabPFN **不需要**特征缩放/标准化，直接使用原始数值
- `BATCH_SIZE` 控制预测分批大小，数据量大时调小防 OOM
- 每个目标独立训练一个模型（非多输出联合模型）
- 模型文件为 zip 归档格式，包含 `init_params.json`、`fitted_attrs.joblib`、`executor_state.joblib`

## 7. SHAP 特征重要性分析

**脚本**：`scripts/analysis/shap_analysis.py`

**作用**：基于 TabPFN 模型计算 SHAP（Shapley Additive Explanations）值，分析各输入特征对输出目标的重要性。

### 7.1 运行命令

```bash
conda run -n tabpfn python scripts/analysis/shap_analysis.py
```

### 7.2 输入文件

| 文件 | 说明 |
|------|------|
| `data/3000_fixed_temp/ml_dataset.csv` | ML 训练数据集 |
| `experiments/3000_fixed_temp/tabpfn/{experiment}/saved_models/` | 已训练的 TabPFN 模型 |

### 7.3 可调参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `DATA_PATH` | `data\3000_fixed_temp\ml_dataset.csv` | 数据文件路径 |
| `TABPFN_DIR` | `experiments\3000_fixed_temp\tabpfn` | TabPFN 实验基目录 |
| `DEVICE` | `cuda` | 计算设备 |
| `TARGETS` | None | None = 全部 29 个目标，或指定列表 |
| `N_EXPLAIN` | 50 | 解释的样本数（越多越慢但越稳定） |
| `BUDGET` | 256 | SHAP 计算预算（16 个特征：65536=精确，256=近似） |
| `EXPERIMENT_DIR` | None | None = 自动选最新实验，或指定如 `'20260622_2247'` |

### 7.4 分析流程

1. 加载数据集
2. 对每个目标：
   - 加载对应的 `.tabpfn_fit` 模型
   - 创建 SHAP imputation explainer（`index='SV'`, `max_order=1`）
   - 对 `N_EXPLAIN` 个样本计算 SHAP 值
   - 生成三种图表：蜂群图、条形图、瀑布图
3. 汇总所有目标的平均 |SHAP| 值，生成重要性矩阵和热力图

### 7.5 输出目录

```
experiments/3000_fixed_temp/tabpfn/{experiment}/shap_results/
├── shap_importance.csv             # SHAP 重要性矩阵（目标 × 特征）
├── shap_heatmap.png                # 特征重要性热力图
├── matte_Cu_pct_summary.png        # 各目标的蜂群图
├── matte_Cu_pct_bar.png            # 各目标的条形图
├── matte_Cu_pct_waterfall.png      # 各目标的瀑布图
├── ...（29 个目标 × 3 种图 = 87 个 PNG 文件）
```

### 7.6 工具函数

可从已有的 `shap_importance.csv` 直接重新生成热力图（无需重跑 SHAP 计算）：

```python
from scripts.analysis.shap_analysis import plot_heatmap_from_csv
plot_heatmap_from_csv('experiments/3000_fixed_temp/tabpfn/20260622_2247/shap_results/shap_importance.csv')
```

### 7.7 注意事项

- SHAP 计算**较慢**，建议先用小 `N_EXPLAIN`（如 10）和小 `BUDGET`（如 64）测试
- 热力图为行归一化（每个目标的重要性除以其最大值），便于跨目标比较
- 若某个目标的模型文件不存在，该目标会被静默跳过
- 背景数据使用 `X[:200]`

## 8. NSGA-II 配矿优化

**脚本**：`scripts/optimization/nsga2_optimization.py`

**作用**：使用 NSGA-II 多目标优化算法，找到最优的矿料配比方案，最小化渣含铜损失，同时满足冶金约束。

### 8.1 运行命令

```bash
conda run -n tabpfn --env PYTHONIOENCODING=utf-8 python scripts/optimization/nsga2_optimization.py
```

**预计耗时**：10~20 分钟（取决于 GPU）

### 8.2 输入文件

| 文件 | 说明 |
|------|------|
| `experiments/3000_fixed_temp/tabpfn/20260622_2247/saved_models/slag_Cu_pct.tabpfn_fit` | 渣含铜预测模型 |
| `experiments/3000_fixed_temp/tabpfn/20260622_2247/saved_models/matte_Cu_pct.tabpfn_fit` | 冰铜品位预测模型 |
| `experiments/3000_fixed_temp/tabpfn/20260622_2247/saved_models/slag_FeO_SiO2_ratio.tabpfn_fit` | 渣 FeO/SiO₂ 预测模型 |

### 8.3 决策变量（12 维）

| 变量 | 范围 | 含义 |
|------|------|------|
| x[0]~x[10] | [0, 1] | 11 种矿料的配比权重（归一化后总和=1） |
| x[11] | [0, 0.15] | SiO₂ 熔剂添加比例（占矿石质量的 0~15%） |

**11 种矿料顺序**：炎鑫、汇祥永金、嘉能可ET、金源矿冶、阿舍勒、路源、外矿BOZ、外矿AKT、和鑫、渣精矿、白银

### 8.4 固定工艺参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `T_DEFAULT` | 1200.0 | 熔炼温度（℃） |
| `P_DEFAULT` | 1.0 | 炉内压强（atm） |
| `O2_DEFAULT` | 21.0 | 供氧量（g） |
| `TOTAL_CHARGE` | 100.0 | 总炉料质量（g） |

### 8.5 优化参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `POP_SIZE` | 100 | 种群大小 |
| `N_GEN` | 200 | 最大迭代代数 |
| `SEED` | 42 | 随机种子 |
| `PATIENCE` | 30 | 早停止耐心值（连续无改善的代数） |

### 8.6 目标函数

**单目标**：最小化 `slag_Cu_pct`（渣含铜 wt%）

渣含铜越低，铜损失越少，冶炼效率越高。

### 8.7 约束条件

**模型预测约束（4 个不等式约束）**：

| 约束 | 公式 | 含义 |
|------|------|------|
| g₁ | FeO/SiO₂ ≥ 1.15 | 渣不能太酸性 |
| g₂ | FeO/SiO₂ ≤ 1.30 | 渣不能太碱性 |
| g₃ | matte Cu ≥ 58 wt% | 冰铜品位不能太低 |
| g₄ | matte Cu ≤ 60 wt% | 冰铜品位不能太高 |

**冶金预过滤约束（不满足则直接惩罚）**：

| 成分 | 下限 | 上限 |
|------|------|------|
| Cu | 14 wt% | 26 wt% |
| S | 8 wt% | 38 wt% |
| Fe | 8 wt% | 36 wt% |
| SiO₂ | 3 wt% | 25 wt% |
| Fe/SiO₂ | 0.8 | 4.0 |

### 8.8 算法配置

- 算法：NSGA-II
- 交叉：SBX（prob=0.9, eta=15）
- 变异：PM（eta=20）
- 采样：FloatRandomSampling
- 去重：eliminate_duplicates=True

### 8.9 输出目录

```
experiments/3000_fixed_temp/optimization/
├── optimization_pareto_all.csv     # 所有 Pareto 最优解
├── optimization_results.csv        # 5 个代表性方案
└── figures/
    ├── fig_pareto_front.png        # Pareto 前沿散点图
    ├── fig_pareto_front.pdf

    └── fig_optimization_comparison.pdf
```

**optimization_pareto_all.csv 列说明**：

| 列名 | 含义 |

### 8.10 注意事项

- Windows 上必须加 `--env PYTHONIOENCODING=utf-8` 避免 GBK 编码错误
- 仅加载 3 个 TabPFN 模型（渣含铜、冰铜品位、渣 FeO/SiO₂），而非全部 29 个
- 人工配矿对比值为硬编码估计值：slag_Cu=0.65, matte_Cu=58.0, FeO/SiO₂=1.35
- 单目标优化时，NSGA-II 退化为类似 GA 的行为，最终返回单一最优解
- 如需多目标优化（如同时最小化渣含铜和最大化冰铜品位），需修改 `n_obj` 和 `_evaluate` 函数

## 10. 数据流水线总览

完整的端到端流程如下：

```
Step 1: 数据生成
    conda run -n tabpfn python scripts/data_processing/generate_samples.py
    输入: data/矿源.xlsx
    输出: data/合成炉料样本.txt

Step 2: FactSage 批量计算（手动）
    在 FactSage 8.3 中加载 data/合成炉料样本.txt
    输出: data/3000_fixed_temp/factsage_results/result_*.txt

Step 3: 结果转 CSV
    conda run -n tabpfn python scripts/data_processing/convert_factsage_to_csv.py
    输入: factsage_inputs.txt + result_*.txt
    输出: data/3000_fixed_temp/ml_dataset.csv

Step 4: TabPFN 模型训练
    conda run -n tabpfn python scripts/training/train_tabpfn.py
    输入: ml_dataset.csv
    输出: experiments/3000_fixed_temp/tabpfn/{timestamp}/saved_models/

Step 5: 基线算法对比（可选）
    conda run -n tabpfn python scripts/analysis/compare_baselines.py
    输入: ml_dataset.csv + TabPFN 模型
    输出: experiments/3000_fixed_temp/baselines/{timestamp}/

Step 6: SHAP 分析（可选）
    conda run -n tabpfn python scripts/analysis/shap_analysis.py
    输入: ml_dataset.csv + TabPFN 模型
    输出: experiments/3000_fixed_temp/tabpfn/{timestamp}/shap_results/

Step 7: 配矿优化
    conda run -n tabpfn --env PYTHONIOENCODING=utf-8 python scripts/optimization/nsga2_optimization.py
    输入: TabPFN 模型（slag_Cu, matte_Cu, FeO/SiO₂）
    输出: experiments/3000_fixed_temp/optimization/

Step 8: 重新生成图表（可选）
    conda run -n tabpfn python scripts/optimization/plot_optimization.py
    输入: optimization CSV 文件
    输出: 更新图表
```

### 依赖关系

```
Step 1 → Step 2（手动）→ Step 3 → Step 4
                                    ↓
                              Step 5（可选）
                              Step 6（可选）
                              Step 7 → Step 8（可选）
```

- Step 5、6 依赖 Step 4 的模型
- Step 7 依赖 Step 4 的模型（仅需 3 个）
- Step 8 依赖 Step 7 的输出
- Step 5、6 与 Step 7 互相独立，可并行运行

## 11. 常见问题

### Q1: 运行时报 `UnicodeEncodeError: 'gbk' codec can't encode character`

Windows 上 `conda run` 默认使用 GBK 编码。解决方法：

```bash
conda run -n tabpfn --env PYTHONIOENCODING=utf-8 python <脚本路径>
```

### Q2: TabPFN 训练时 OOM（显存不足）

减小 `BATCH_SIZE` 参数（如从 1000 改为 500 或 200）。

### Q3: SHAP 计算太慢

减小 `N_EXPLAIN`（如 50→20）和 `BUDGET`（如 256→64）。正式分析时再调大。

### Q4: 基线对比的随机种子不一致

`compare_baselines.py` 的 `RANDOM_STATE` 和 `TEST_SIZE` 必须与 `train_tabpfn.py` 一致，否则使用不同的测试集，对比不公平。

### Q5: 优化结果只有 1 个解

单目标优化（1 个目标）的最优解唯一，这是正常的。如需多个权衡方案，需添加第二个目标（如最大化冰铜品位），使 `n_obj=2`。

### Q6: 如何切换数据集

修改各脚本顶部的 `DATA_PATH` 参数，指向新的 `ml_dataset.csv` 文件。同时更新 `RESULTS_DIR` 以避免结果混在一起。

### Q7: FactSage 结果文件编码问题

FactSage 输出文件通常为 UTF-8 编码。若解析失败，检查文件编码并修改 `convert_factsage_to_csv.py` 中的 `encoding` 参数。

### Q8: 如何在不同温度下重新实验

1. 修改 `generate_samples.py` 中的 `TEMP_FIXED`（或 `TEMP_MIN`/`TEMP_MAX`）
2. 重新生成样本
3. 重新运行 FactSage 计算
4. 将结果放入新的数据集目录（如 `data/1300_fixed_temp/`）
5. 修改各脚本的 `DATA_PATH` 指向新数据集
